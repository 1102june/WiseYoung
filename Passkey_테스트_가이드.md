# Passkey 테스트 가이드

## ⚠️ 에뮬레이터/포트포워딩 기기 제한사항

Android 에뮬레이터나 일부 포트포워딩된 기기에서는 **Passkey를 생성하거나 사용할 수 없습니다**.

### 이유
- Passkey는 하드웨어 보안 기능(지문, 얼굴 인식 등)이 필요합니다
- 에뮬레이터는 이러한 하드웨어 기능을 완전히 시뮬레이션할 수 없습니다
- 일부 기기는 보안 정책으로 Passkey를 지원하지 않을 수 있습니다

## ✅ 해결 방법

### 1. 실제 기기 사용 (권장)

**가장 확실한 방법입니다.**

1. **USB 디버깅 활성화**
   - 개발자 옵션 > USB 디버깅 활성화
   - PC와 USB로 연결

2. **앱 설치 및 테스트**
   - Android Studio에서 실제 기기 선택
   - 앱 실행 및 Passkey 테스트

3. **네트워크 설정**
   - PC와 기기가 같은 Wi-Fi에 연결
   - PC의 로컬 IP 주소 사용 (예: `http://192.168.0.100:8080`)

### 2. 개발 모드에서 Passkey 우회 (임시)

개발 중에는 Passkey 없이 테스트할 수 있는 옵션을 사용할 수 있습니다.

**주의:** 프로덕션에서는 반드시 실제 Passkey를 사용해야 합니다.

### 3. 에뮬레이터에서 테스트 (제한적)

일부 최신 에뮬레이터에서는 Passkey를 지원할 수 있지만, 완전하지 않을 수 있습니다.

## 📱 실제 기기 테스트 설정

### PC 설정
1. **로컬 IP 주소 확인**
   ```powershell
   ipconfig
   # IPv4 주소 확인 (예: 192.168.0.100)
   ```

2. **방화벽 설정**
   - Windows 방화벽에서 포트 8080 허용
   - 또는 방화벽 일시 비활성화 (개발용)

3. **백엔드 서버 실행**
   ```powershell
   cd C:\WiseYoung_backend
   .\run-local.ps1
   ```

### Android 기기 설정
1. **같은 Wi-Fi 연결**
   - PC와 기기가 같은 Wi-Fi 네트워크에 연결

2. **앱 설정**
   - `NetworkModule.kt`에서 BASE_URL을 PC의 로컬 IP로 변경
   - 예: `http://192.168.0.100:8080/`

3. **앱 실행 및 테스트**
   - Passkey 등록 및 로그인 테스트

## 🔧 개발 모드 옵션 (선택사항)

개발 중에는 Passkey 없이 테스트할 수 있는 옵션을 추가할 수 있습니다.

**구현 방법:**
- `BuildConfig.DEBUG`를 사용하여 개발 모드 감지
- 개발 모드에서는 Passkey 대신 간단한 인증 사용
- 또는 Passkey 오류 시 자동으로 대체 인증 사용

## 🚨 문제 해결

### "Passkey를 생성할 수 없습니다" 오류
- **원인:** 에뮬레이터 또는 Passkey 미지원 기기
- **해결:** 실제 기기 사용

### "서버 연결 실패" 오류
- **원인:** 네트워크 설정 문제
- **해결:** 
  - PC와 기기가 같은 Wi-Fi에 연결되어 있는지 확인
  - PC의 로컬 IP 주소가 올바른지 확인
  - 방화벽 설정 확인

### "등록된 Passkey가 없습니다" 오류
- **원인:** Passkey가 등록되지 않음
- **해결:** 먼저 Passkey 등록 기능 사용

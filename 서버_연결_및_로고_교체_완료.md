# 서버 연결 및 앱 로고 교체 완료

## ✅ 완료된 작업

### 1. 서버 연결 문제 해결
- **ADB 포트 포워딩 설정 완료**: `adb reverse tcp:8080 tcp:8080`
- **서버 상태 확인**: Spring Boot 서버가 8080 포트에서 정상 실행 중
- **연결 방법**: `http://127.0.0.1:8080` (ADB 포트 포워딩 사용)

### 2. 앱 로고 교체
- **drawable/wy_logo.png**: 교체 완료
- **mipmap 폴더의 wy_logo.png**: 모든 해상도 교체 완료
- **wy_logo_foreground.png**: 모든 해상도 교체 완료

## 🔧 다음 단계

### 서버 연결 확인
1. **Spring Boot 서버가 실행 중인지 확인**
   ```powershell
   netstat -an | findstr :8080
   ```
   - `LISTENING` 상태여야 함 ✅

2. **ADB 포트 포워딩 확인**
   ```powershell
   adb reverse --list
   ```
   - `tcp:8080 tcp:8080`이 표시되어야 함 ✅

3. **앱 재빌드 및 실행**
   - 앱을 다시 빌드하고 실행하면 서버 연결이 정상적으로 작동합니다.

### 앱 로고 확인
1. **앱 재빌드 필요**
   - 리소스 파일을 변경했으므로 Clean Build를 권장합니다:
     ```
     Build > Clean Project
     Build > Rebuild Project
     ```

2. **앱 아이콘 확인**
   - 앱을 재설치한 후 홈 화면에서 아이콘 확인
   - 앱 내부 로고는 SplashActivity나 다른 화면에서 확인

## ⚠️ 참고사항

### ADB 포트 포워딩
- USB 연결이 끊기면 포트 포워딩이 해제됩니다
- USB를 다시 연결할 때마다 다음 명령어를 실행해야 합니다:
  ```powershell
  adb reverse tcp:8080 tcp:8080
  ```

### 앱 로고
- Adaptive Icon을 사용하는 경우, foreground와 background가 모두 필요합니다
- 현재 설정: `wy_logo_foreground.png` + `wy_logo_background` 색상
- PNG 파일이 제대로 표시되지 않으면 .webp로 변환하거나 Image Asset Studio를 사용하세요

## 📝 문제 해결

### 서버 연결이 여전히 안 되는 경우
1. Spring Boot 서버가 실행 중인지 확인
2. ADB 포트 포워딩이 설정되어 있는지 확인 (`adb reverse --list`)
3. USB 디버깅이 활성화되어 있는지 확인
4. 기기를 다시 연결하고 포트 포워딩 재설정

### 앱 로고가 변경되지 않는 경우
1. Clean Build 실행
2. 앱 완전 삭제 후 재설치
3. Android Studio의 Image Asset Studio 사용 (권장)
